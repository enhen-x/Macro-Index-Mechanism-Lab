# MacroDynamics-System-ID Roadmap

## 目标

在可复现的数据与实验框架上，完成“指数动力系统建模 -> 参数辨识 -> 验证评估 -> 自适应升级”的闭环。

## 阶段计划（2026）

## Phase 1: 数据管线（2026-03 到 2026-04）

目标：
- 打通价格与宏观数据的统一输入格式。
- 建立基础数据质量控制（时间对齐、缺失处理、异常值标记）。

交付物：
- `data_loader/` 基础数据加载模块
- 最小样本数据生成器（用于无外部数据时的验证）
- 数据字典文档（字段、频率、单位）

验收标准：
- 同一实验可重复运行并得到一致结果
- 样本数据可直接驱动后续辨识模块

## Phase 2: 基线辨识（2026-04 到 2026-05）

目标：
- 完成二阶受迫阻尼系统的基线参数辨识。
- 形成首个可解释参数输出：`m/c/k` 与外部驱动系数。

交付物：
- `identification/` 基线 OLS/最小二乘辨识器
- 参数估计报告模板（含误差和稳定性统计）
- 最小回测脚本（train/test 切分）

验收标准：
- 在样本数据上可输出稳定参数估计
- 具备基础 out-of-sample 指标（MSE/方向性命中率）

## Phase 3: 自适应建模（2026-05 到 2026-07）

目标：
- 引入时变参数机制（如 EKF 或 rolling window）。
- 识别并量化状态切换下参数漂移。

交付物：
- 自适应辨识器（rolling / EKF 二选一先落地）
- 状态切换诊断报告（参数漂移曲线）
- 评估脚本升级（按状态分桶评估）

验收标准：
- 对比 Phase 2，样本外指标有可解释改进
- 参数漂移与宏观冲击有可追溯对应关系

## Phase 4: 物理约束 ML 融合（2026-07 到 2026-09）

目标：
- 将物理约束引入数据驱动模型（PINNs 或混合损失）。
- 对比纯统计模型与物理约束模型的泛化能力。

交付物：
- 物理约束模型实验分支
- 对照实验报告（统计模型 vs 物理约束模型）
- 失败案例库（何时物理约束不提升）

验收标准：
- 至少一个市场/区间表现优于基线且具可解释性
- 模型局限性被明确记录并可复现

## 横向治理项（全周期）

- 实验可复现：固定随机种子、统一配置、结果落盘
- 工程质量：最小单元测试、基础 lint、代码风格统一
- 文档规范：每个阶段完成时更新 README 与 docs

## 里程碑检查点

- M1（2026-04-15）：数据管线可稳定供数
- M2（2026-05-20）：基线辨识结果可复现
- M3（2026-07-01）：自适应模型进入评估
- M4（2026-09-01）：完成物理约束融合对照报告
